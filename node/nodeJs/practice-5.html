# 5-nodeJs文件系统fs模块常用api

引入文件系统模块

const fs = require('fs')
1

文件系统类下的属性和方法

# 1. fs.readFile(path[, options], callback)

读取文件

const fs = require('fs')
fs.readFile('./hello.txt', (err, data) => {
    if (err) throw err;
    console.log(data)
    // <Buffer 68 65 6c 6c 6f 20 77 6f 72 6c 64>
})

通过文件系统fs读取的文件返回的数据是二进制格式，我们可以通过buf实例的toString方法，将buf解码成字符串

const fs = require('fs')
fs.readFile('./hello.txt', (err, data) => {
    if (err) throw err;
    console.log(data.toString())
    // hello world
})

# 5. fs.stat(path[, options], callback)

获取文件信息，判断是文件还是文件夹

fs.stat('./hello.txt', (err, stats) => {
    if (err) throw err;
    console.log(stats)
})
1
2
3
4

根据返回的stats这个对象，它有很多属性和方法，我们可以利用这些方法做一些操作

fs.stat('./hello.txt', (err, stats) => {
    if (err) {
        console.log('文件不存在')
        return;
    };
    console.log(stats)
    console.log(stats.isFile()) // true 是否是文件
    console.log(stats.isDirectory()) // false 是否是文件夹
})

# 11. fs.watch(filename[, options][, listener])

监听文件的变化

fs.watch('./', (eventType, filename) => {
    console.log(eventType, filename)
})
1
2
3

同时也可以在options里设置recursive来监视子目录的变化

fs.watch('./', {
    recursive: true
}, (eventType, filename) => {
    console.log(eventType, filename)
})
1
2
3
4
5

利用fs文件系统提供的watch这个方法，输出的时候，改动一下，会同时打印多次，效果不是特别理想，所以我们可以利用chokidar这个插件来监听文件的变化

chokidar 在需要安装此插件的文件执行如下命令：

npm init -y
1
npm/cnpm install chokidar --save-dev
1

使用代码如下：

const chokidar = require('chokidar')
chokidar.watch('./').on('all', (event, files) => {
    console.log(event, files);
})
1
2
3
4

chokidar还有一些配置，介绍如下：

const chokidar = require('chokidar')
chokidar.watch('./', {
    ignored: './node_modules', // 不监听node_modules
}).on('all', (event, files) => {
    console.log(event, files);
})
